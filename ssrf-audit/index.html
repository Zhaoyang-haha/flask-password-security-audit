一、实验概述

Experiment Overview

1.1 什么是 SSRF？

SSRF（服务端请求伪造） 是一种攻击方式，攻击者利用服务端提供的 URL 访问功能，诱使服务端向内网或本地系统发起请求，从而访问或攻击本不应被外部访问的内部资源。

攻击者控制 URL 参数 → 目标服务器（Flask） ├─ file:///etc/passwd → 读取本地文件 ├─ http://127.0.0.1:5000 → 内网访问（本机服务） ├─ http://192.168.1.1/ → 内网渗透 └─ http://169.254.169.254/→ 云元数据攻击

二、漏洞分析与修复

Vulnerability Analysis & Remediation

1

任意协议 SSRF（file:// 读取本地文件）

严重 · CVSS 9.0

CWE-918 · 无协议限制，urllib 支持 file:// 等协议

Python — 问题代码
url = request.form.get("url", "")
req = urllib.request.Request(url)          # 任何协议都接受
response = urllib.request.urlopen(req)      # file:// 也能打开！

攻击复现：curl -X POST ... -d "url=file:///etc/passwd" → 读取系统密码文件；还可读取 app.py 源码、data/users.db 数据库。

❌ 修复前无协议限制，支持 file://
✅ 修复后仅允许 http:// 和 https://
2

内网 IP SSRF（127.0.0.1 访问）

严重 · CVSS 8.6

CWE-918 · 无内网 IP 拦截，可访问本机服务

攻击复现：curl ... -d "url=http://127.0.0.1:5000/" → 访问本机 Flask 服务。攻击者可探测内网 Redis(6379)、MySQL(3306) 等敏感端口。

3

内网段 SSRF（192.168.x.x / 10.x.x.x）

严重

无内网范围限制，可探测整个内网段

for ip in 192.168.14.{1..254}; do
  for port in 22 80 443 3306 6379 8080; do
    curl -X POST ... -d "url=http://$ip:$port/" &
  done
done

攻击者可通过 SSRF 对内网进行横向移动，探测存活主机和开放端口。

4

域名解析绕过 SSRF（localhost）

严重

仅字符串匹配 IP 无法防御域名绕过

http://localhost/ → 127.0.0.1 http://127.1/ → 127.0.0.1（简写） http://0x7f000001/ → 十六进制 http://2130706433/ → 十进制 http://0177.0.0.1/ → 八进制

修复：使用 socket.gethostbyname() 解析域名后检查真实 IP 是否为内网。

5

无 CSRF 防护

中危

CWE-352 · URL 抓取接口无 CSRF Token 校验

攻击者可构造 CSRF+SSRF 组合攻击：诱导管理员访问恶意页面 → 浏览器自动提交 POST 请求 → 服务端以管理员身份执行 SSRF。

三、SSRF 防护方案

Security Hardening

3.1 六层纵深防御架构

用户提交 URL ↓ 第1关：CSRF Token 校验 → 403 ↓ 第2关：URL 长度限制（≤2048） ↓ 第3关：协议白名单（仅 http/https） ↓ 第4关：DNS 解析 + IP 黑白名单 ↓ 第5关：安全请求（timeout=10） ↓ 第6关：统一错误信息

3.2 核心修复代码

Python
def is_internal_ip(hostname):
    try:
        ip = socket.gethostbyname(hostname)
        if ip.startswith("127.") or ip.startswith("10."): return True
        if ip.startswith("169.254."): return True
        if ip.startswith("192.168."): return True
        if ip == "0.0.0.0": return True
        return False
    except: return True

@app.route("/fetch-url", methods=["POST"])
@login_required
def fetch_url():
    if not validate_csrf(): abort(403)           # 第1关
    url = request.form.get("url", "")
    if len(url) > 2048: return error("URL过长")  # 第2关
    if not url.startswith(("http://","https://")):  # 第3关
        return error("不支持的协议")
    parsed = urllib.parse.urlparse(url)
    if is_internal_ip(parsed.hostname):           # 第4关
        return error("不允许访问内网地址")
    try:
        req = urllib.request.Request(url)
        with urllib.request.urlopen(req, timeout=10) as r:
            content = r.read().decode("utf-8", errors="replace")
            result = {"status_code": r.getcode(), "content": content[:5000]}
    except: error = "请求失败，请稍后重试"          # 第6关

四、总结与项目汇总

Summary & Project Overview

4.1 SSRF = 服务端的信任危机

第1层：CSRF Token（防止 SSRF 被 CSRF 利用） 第2层：URL 长度限制（防止资源耗尽） 第3层：协议白名单（阻止 file:// 等危险协议） 第4层：DNS 解析验证（防止域名绕过） 第5层：IP 黑白名单（阻止内网地址） 第6层：统一错误提示（防止信息泄露）
