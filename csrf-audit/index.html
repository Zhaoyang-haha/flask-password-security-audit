一、实验概述

Experiment Overview

1.1 什么是 CSRF？

跨站请求伪造（CSRF） 是一种攻击方式，攻击者诱导已登录用户访问恶意页面，在该页面中自动发起对目标网站的请求，利用浏览器自动携带 Cookie 的机制，冒充用户身份执行非本意操作。

1.3 CSRF 攻击三大条件

条件1：用户已登录目标站点（浏览器存有 Session Cookie） 条件2：目标站点的 POST 请求无额外校验（无 CSRF Token） 条件3：用户访问了攻击者控制的页面（自动提交表单）

二、漏洞分析与修复

Vulnerability Analysis & Remediation

1

登录接口 CSRF 缺失

中危

CWE-352 · POST /login

攻击者构造恶意页面： <form action="http://target/login" method="POST" style="display:none"> <input name="username" value="attacker"> <input name="password" value="hacked123"> </form><script>document.forms[0].submit()</script> 当受害者访问 → 浏览器自动提交 → 攻击者获得登录 Session
2

注册接口 CSRF 缺失

中危

CWE-352 · POST /register — 跨站注册创建恶意账号

3

上传接口 CSRF 缺失

中危

CWE-352 · POST /upload — 跨站上传恶意文件

4

充值接口 CSRF 缺失

高危

CWE-352 · POST /recharge

攻击者构造： <form action="http://target/recharge" method="POST" style="display:none"> <input name="user_id" value="2"> ← 攻击者ID <input name="amount" value="10000"> ← 金额 </form> 攻击流程：受害者访问恶意页面 → 浏览器自动提交POST → 携带受害者的 Cookie → 攻击者账户收到10000元！ 结合负值充值漏洞（amount未校验正负）→ amount=-50000 → 从受害者账户扣钱
5

修改密码接口 CSRF 缺失

高危

CWE-352 · POST /change-password（本轮新增）

严重性：CSRF 缺失 + IDOR 越权 = 任意用户密码可被跨站修改。无 CSRF Token + 无 IDOR 校验 + 无原密码校验 = 攻击者只需让管理员访问一个恶意页面，管理员密码即被修改为攻击者指定的值！
Python — 修复方案
@app.route("/change-password", methods=["POST"])
@login_required
def change_password():
    # CSRF 校验
    if not validate_csrf():
        abort(403, "CSRF 验证失败")
    # IDOR 校验（从 session 获取用户名）
    session_username = session.get("username", "")
    username = request.form.get("username", "")
    if username != session_username:
        abort(403, "无权修改其他用户的密码")

三、CSRF 防护方案

Security Hardening

3.1 三层防护体系

第一层（核心）：CSRF Token（Synchronizer Token Pattern） 每个表单嵌入随机 Token，服务端校验匹配 第二层（辅助）：SameSite Cookie Cookie 设置 SameSite=Lax，阻止跨站 POST 携带 Cookie 第三层（进阶）：一次性 Token（防重放） 成功操作后刷新 Token，旧 Token 立即失效

3.2 核心代码实现

Python
import secrets

def generate_csrf_token():
    if "csrf_token" not in session:
        session["csrf_token"] = secrets.token_hex(16)
    return session["csrf_token"]

def validate_csrf():
    token = request.form.get("csrf_token", "")
    stored = session.get("csrf_token", "")
    return bool(token and token == stored)

def refresh_csrf():
    session["csrf_token"] = secrets.token_hex(16)

@app.before_request
def ensure_csrf():
    if "csrf_token" not in session:
        session["csrf_token"] = secrets.token_hex(16)

3.3 防重放机制

用户 GET /profile → Token = "abc123" 用户 POST /recharge + Token="abc123" → 校验通过 ✅ → 执行充值 ✅ → 刷新 Token 为 "def456" 攻击者重放 POST /recharge + Token="abc123" → 校验失败 ❌（Token 已被刷新，旧 Token 失效）

"Cookie 是浏览器对用户的承诺，但不是对他人意图的保证。"

CSRF 漏洞不需要破解密码、不需要复杂的 payload，只需要一个 HTML 表单和用户无意间的一次点击。防御方式同样简单：每个 POST 请求都必须有一个不可预测的 Token，且这个 Token 只能使用一次。
